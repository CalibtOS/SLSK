# Prompt: Odoo Module Interaction Diagram

---

## Prompt

Create a full-width, interactive HTML diagram showing all major Odoo 17/18 modules and how they interact with each other. The diagram should be visually polished, modern, and suitable for a business presentation.

---

### Visual Requirements

- **Dark background**: `#0F172A` (deep navy)
- **Layout**: Force-directed or hand-placed node graph, centered on screen, filling the full viewport
- **Module nodes**: Rounded rectangle cards (`border-radius: 12px`), each showing the module name and a small icon or symbol
- **Connections**: Animated directional arrows between nodes using SVG `<line>` or `<path>` with arrowhead markers; use a subtle animated dash flow (`stroke-dashoffset` animation) to show data movement direction
- **Color coding by category** (use distinct saturated colors per group):
  - 🔵 **Sales & Commerce**: `#3B82F6` (blue) — CRM, Sales, eCommerce, Point of Sale, Subscriptions
  - 🟠 **Purchase**: `#F97316` (orange) — Purchase, Vendor Portal
  - 🟢 **Inventory & Warehouse**: `#10B981` (emerald) — Inventory, Delivery, Receipts, Barcode
  - 🟣 **Manufacturing**: `#8B5CF6` (purple) — Manufacturing, BoM, Work Centers, Quality, Maintenance, PLM
  - 🔴 **Accounting & Finance**: `#EF4444` (red) — Accounting, Invoicing, Expenses, Payroll
  - 🟡 **Human Resources**: `#F59E0B` (amber) — Employees, Recruitment, Attendance, Leaves, Appraisals
  - 🩵 **Project & Services**: `#06B6D4` (cyan) — Project, Timesheets, Field Service, Helpdesk
  - ⚪ **Core / Base**: `#94A3B8` (slate) — Contacts, Settings, Users, Discuss
  - 🩷 **Marketing**: `#EC4899` (pink) — Email Marketing, Marketing Automation, SMS, Social
- **Node size**: Larger nodes for central/high-connectivity modules (Inventory, Accounting, Sales), smaller for peripheral ones
- **Hover interaction**: On hover, highlight the hovered node and all its direct connections; dim everything else
- **Tooltip on hover**: Show the module's full name and a one-line description of what it does
- **Legend**: Fixed bottom-left panel showing each category color + name
- **Title**: Top-left corner — "Odoo ERP · Module Architecture" in white, 18px bold

---

### Modules to Include

Group them in clusters on the canvas (loosely by category, but allow cross-category arrows):

#### Core / Base
- **Contacts** (res.partner — master record shared by all modules)
- **Settings** (multi-company, users, access rights)
- **Discuss** (internal chat & notifications)

#### Sales & Commerce
- **CRM** → generates → **Sales Order**
- **Sales** → triggers → **Delivery**, **Invoice**
- **eCommerce** → feeds → **Sales**, **Inventory**
- **Point of Sale** → syncs → **Inventory**, **Accounting**
- **Subscriptions** → generates recurring → **Sales**, **Invoicing**

#### Purchase
- **Purchase** → triggers → **Receipts (Inventory)**, **Vendor Bill (Accounting)**
- **Purchase** ← replenishment trigger ← **Inventory**
- **Purchase** ← auto-PO ← **Manufacturing**

#### Inventory & Warehouse
- **Inventory** ↔ **Manufacturing** (component consumption, finished goods receipt)
- **Inventory** → **Accounting** (stock valuation journal entries)
- **Inventory** → **Delivery** (outgoing shipments from Sales)
- **Inventory** → **Receipts** (incoming from Purchase)
- **Barcode** → scan interface → **Inventory**

#### Manufacturing
- **Manufacturing** → consumes → **Inventory**
- **Manufacturing** → produces → **Inventory**
- **Manufacturing** ← BoM — **Bill of Materials**
- **Work Centers** ← scheduled by — **Manufacturing**
- **Quality** ← triggered at checkpoints — **Manufacturing**, **Inventory**
- **Maintenance** ← triggered by — **Work Centers**
- **PLM** → version control → **BoM**

#### Accounting & Finance
- **Accounting** ← journal entries ← **Inventory**, **Payroll**, **Invoicing**, **Expenses**, **Point of Sale**
- **Invoicing** ← generated by ← **Sales**, **Project/Timesheets**, **Subscriptions**
- **Expenses** → posted to → **Accounting**
- **Payroll** → journal entries → **Accounting**

#### Human Resources
- **Employees** → base record → **Payroll**, **Attendance**, **Leaves**, **Expenses**, **Appraisals**
- **Recruitment** → creates → **Employees**
- **Attendance** → feeds → **Payroll**

#### Project & Services
- **Project** → tracks → **Timesheets**
- **Timesheets** → billable → **Invoicing**
- **Field Service** → generates → **Timesheets**, **Invoicing**
- **Helpdesk** → escalates to → **Field Service**, **Project**

#### Marketing
- **CRM** ← leads generated by ← **Email Marketing**, **Marketing Automation**
- **eCommerce** ← driven by ← **Social**, **Email Marketing**

---

### Connection Styling Rules

- Solid animated arrow = **data/document flow** (e.g. Sales Order → Delivery)
- Dashed animated arrow = **trigger/signal** (e.g. reorder point → Purchase)
- Arrow color = **source module's category color**
- Arrow width = 1.5–2px, arrowhead size = 8px

### Technical Implementation

- Pure HTML + SVG (no external libraries except optionally D3.js for force layout)
- All in a single `.html` file
- SVG `<defs>` for arrowhead markers per category color
- Use `requestAnimationFrame` or CSS `@keyframes` for the flow animations
- Responsive: fills `100vw × 100vh`
- Smooth zoom + pan: mouse wheel to zoom, drag to pan (transform on the SVG `<g>` container)
- Node positions can be hardcoded (preferred for clarity) or force-simulated
- On node click: open a small modal/sidebar with: module name, category, description, list of connected modules

---

### Deliverable

A single, self-contained `odoo_diagram.html` file that opens in any modern browser and renders the full Odoo module interaction diagram as described above.
